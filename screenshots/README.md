# Інструкції для створення скріншотів

Для завершення лабораторної роботи необхідно зробити наступні скріншоти:

## 1. home.png
**Що показати:** Головна сторінка (Dashboard)
- Відкрийте `http://localhost:5173/`
- Переконайтесь, що видно:
  - Hero секцію з назвою "Система управління бібліотекою LibraryHub"
  - Статистику (4 картки з цифрами)
  - Можливості системи (6 карток з іконками)
- Зробіть скріншот всієї сторінки

## 2. books-list.png
**Що показати:** Список книг з пошуком
- Відкрийте `http://localhost:5173/books`
- Переконайтесь, що:
  - Видно список книг (картки з інформацією)
  - Поле пошуку вгорі сторінки
  - Кнопки "Редагувати" та "Видалити" на кожній картці
  - Кнопка "Додати книгу" вгорі
- Зробіть скріншот

## 3. create-book.png
**Що показати:** Форма створення книги (порожня)
- Відкрийте `http://localhost:5173/books/create`
- Переконайтесь, що видно всі поля:
  - Назва книги
  - Автор
  - Видавництво
  - Рік
  - Мова
  - ISBN
  - Локація
  - Кількість
  - Опис
- Кнопку "Створити книгу"
- Зробіть скріншот

## 4. form-validation.png
**Що показати:** Валідаційні помилки
- Відкрийте `http://localhost:5173/books/create`
- **НЕ заповнюйте жодне поле**
- Натисніть кнопку "Створити книгу"
- Зачекайте, поки з'являться червоні повідомлення про помилки під полями
- Переконайтесь, що видно:
  - Червоні border на полях з помилками
  - Червоні текстові повідомлення під полями
  - Різні типи валідації (обов'язкові поля, мінімальна довжина)
- Зробіть скріншот

## 5. edit-book.png
**Що показати:** Форма редагування з даними
- Відкрийте `http://localhost:5173/books`
- Натисніть "Редагувати" на будь-якій книзі
- Переконайтесь, що:
  - Форма заповнена даними існуючої книги
  - Видно всі поля (включаючи language та location)
  - Є секція "Loan History" (якщо є видачі)
  - Кнопки "Зберегти зміни" та "Скасувати"
- Зробіть скріншот

## 6. network-requests.png
**Що показати:** Network Tab з API запитами
- Відкрийте Chrome DevTools (F12)
- Перейдіть на вкладку "Network"
- Відкрийте `http://localhost:5173/books`
- Дочекайтесь завантаження книг
- Знайдіть запит `GET /books` в списку
- Клікніть на нього
- Переконайтесь, що видно:
  - Status code: 200
  - Request Headers з `Authorization: Bearer ...`
  - Response з JSON даними книг
- Зробіть скріншот вкладки Network

**Додатково (опціонально):**
- Створіть книгу і покажіть `POST /books` (201)
- Видаліть книгу і покажіть `DELETE /books/:id` (204)

## 7. react-query-devtools.png
**Що показати:** TanStack Query Devtools
- Відкрийте `http://localhost:5173/books`
- Знайдіть іконку TanStack Query Devtools в лівому нижньому куті
- Клікніть на неї для відкриття devtools
- Переконайтесь, що видно:
  - Query keys: `["books"]`, `["books", "1"]` (якщо відкривали книгу)
  - Стани запитів: fresh, stale, fetching
  - Cached data
  - Query Explorer зліва
- Зробіть скріншот

## 8. login.png
**Що показати:** Форма авторизації
- Відкрийте `http://localhost:5173/login`
- Переконайтесь, що видно:
  - Декоративну шапку з іконкою замка
  - Поля Email та Password
  - Кнопку "Увійти"
  - Дизайн форми
- **Для показу валідації:** введіть невірний email (без @) і натисніть "Увійти"
- Зробіть скріншот з помилками валідації

## 9. mobile-responsive.png
**Що показати:** Мобільна версія
- Відкрийте Chrome DevTools (F12)
- Натисніть кнопку "Toggle device toolbar" (Ctrl+Shift+M)
- Виберіть iPhone або інший мобільний пристрій
- Відкрийте `http://localhost:5173/books`
- Переконайтесь, що:
  - Список книг відображається в один стовпчик
  - Навігація адаптована під мобільний
  - Все читабельно та зручно
- Зробіть скріншот

---

## Важливо:

- Всі скріншоти повинні бути чіткими та читабельними
- Рекомендований формат: PNG
- Назви файлів мають точно відповідати тим, що вказані вище
- Збережіть скріншоти в цю папку (`screenshots/`)

## Як зробити скріншот:

### Windows:
- **Win + Shift + S** - вибрати область екрану
- Або використайте **Snipping Tool**

### macOS:
- **Cmd + Shift + 4** - вибрати область
- **Cmd + Shift + 3** - весь екран

### Linux:
- **PrtScn** або використайте **GNOME Screenshot**

---

Після створення всіх скріншотів, переконайтесь, що всі файли знаходяться в папці `screenshots/`.
